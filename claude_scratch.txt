# Sella Performance Optimization Notes

## Current State (consolidated-optimizations branch)
- 13 commits ahead of origin/master
- Performance: 6.6s vs 19.1s on origin/master (2.9x speedup, 65% reduction)

## Key Optimizations Made
1. Cache SparseInternalHessians - avoid reconstructing sparse hessian objects
2. Cache cell transformations (tvecs) - cache ncvecs @ cell matrix multiplications
3. Direct HVP computation - use jvp(grad(f)) instead of forming full Hessians
4. Reuse Binv across ODE steps - avoid repeated SVD during ODE integration
5. Cache ndof/natoms/ndummies properties - avoid 26k+ redundant len() calls
6. QuasiNewton as default method - avoid expensive RFO eigendecompositions
7. Lazy eigendecomposition in ApproximateHessian - only compute when accessed
8. Share SVD between _calc_basis and _get_Binv - avoid duplicate SVD calls

## Code Cleanup
- Replaced verbose if/else blocks with conditional expressions
- Restructured trust radius update to use proper if/else instead of if X: pass elif
- Cache P.B is None check to avoid duplicate evaluation
- Pass computed basis from _update() to _update_basis() to avoid recomputing
- Extract _ensure_eigen_computed() helper to consolidate eigendecomposition logic
- Vectorize O(n²) loop in SparseInternalHessian.asarray() using np.add.at
